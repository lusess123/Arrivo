# 数据库放通执行清单

## 当前结论

Arrivo 的 Cloudflare 改造已经部署到线上，但数据库网络仍未打通。

当前数据库地址来自 `packages/arrivo-server/.env`：

- host: `111.229.179.154`
- port: `5433`
- database: `arrivo`
- user: `sa`

已验证：

- 本机访问 `111.229.179.154:5433` 超时。
- Cloudflare Worker 访问 `/api/health/db` 超时后返回 500。
- Worker 日志错误为 `proxy request failed, cannot connect to the specified address`。
- `111.229.179.154` 只有 SSH 22 端口能从本机连通，`80/443/3000/3025/3026/3027/5432/5433` 都超时。
- 当前本机已有 SSH key 组合无法登录该机器。
- 当前阿里云 RAM 子账号没有 RDS/ECS 查询权限。
- 本机没有可用腾讯云 CLI 凭证。
- CloudBase 本地只有登录态结构，不能直接操作这台 CVM/数据库。
- Cloudflare Tunnel 私网连接需要在能访问数据库的网络内运行 `cloudflared`。当前没有 SSH 或云控制台入口，无法部署 Tunnel。

## 推荐路径

优先使用 Hyperdrive 连接现有 PostgreSQL：

1. 确认 `111.229.179.154:5433` 确实是当前线上 PostgreSQL。
2. 在数据库所在云厂商安全组中放通 PostgreSQL 端口 `5433`。
3. 在系统防火墙中放通 `5433`。
4. 确认 PostgreSQL 监听公网或正确网卡。
5. 从本机验证 TCP 和 PostgreSQL 登录。
6. 创建 Cloudflare Hyperdrive。
7. 将 Hyperdrive binding 写入 `apps/api/wrangler.jsonc`。
8. 重新部署 API。
9. 执行登录、文章、MDD、TTS 线上端到端验证。

如果数据库不能直接公网放通，可以改用 Cloudflare Tunnel 私网方案，但必须先拿到数据库所在机器或同 VPC 机器的登录权限，在那里运行 `cloudflared`，并把 Tunnel public hostname 的 TCP service 指向 `localhost:5433` 或内网 PostgreSQL 地址。

## 云安全组建议

如果这台机器是腾讯云 CVM：

- 入站规则放通 TCP `5433`。
- 来源先使用 Cloudflare IPv4/IPv6 段。
- 如果排查阶段需要临时验证，也可以短时间放通当前本机公网 IP，验证后收回。

如果这台机器是阿里云 ECS/RDS：

- ECS 安全组入站规则放通 TCP `5433`。
- RDS 白名单增加 Cloudflare IP 段，或临时增加本机公网 IP 做验证。

## 机器内检查命令

登录数据库所在机器后执行：

```bash
ss -ltnp | grep -E ':(5432|5433)'
```

如果 PostgreSQL 未监听 `0.0.0.0:5433` 或公网网卡，需要检查：

```bash
sudo grep -nE '^(listen_addresses|port)' /etc/postgresql/*/*/postgresql.conf
sudo grep -nE 'host.*0\\.0\\.0\\.0|host.*::/0' /etc/postgresql/*/*/pg_hba.conf
```

常见修复方向：

```bash
# postgresql.conf
listen_addresses = '*'
port = 5433

# pg_hba.conf，根据实际安全策略限制来源
host    arrivo    sa    0.0.0.0/0    md5
host    arrivo    sa    ::/0         md5
```

重启 PostgreSQL：

```bash
sudo systemctl restart postgresql
```

如果使用 Docker 运行 PostgreSQL，确认端口映射：

```bash
docker ps --format 'table {{.Names}}\t{{.Ports}}\t{{.Image}}'
```

## Cloudflare Hyperdrive

数据库公网访问打通后，创建 Hyperdrive：

```bash
cd apps/api
bunx wrangler hyperdrive create arrivo-postgres-prod --connection-string "$DATABASE_URL"
```

把返回的 id 写入 `apps/api/wrangler.jsonc`：

```jsonc
"hyperdrive": [
  {
    "binding": "HYPERDRIVE",
    "id": "<hyperdrive-id>"
  }
]
```

重新部署：

```bash
bun run api-deploy
```

线上验证：

```bash
curl -i https://api-arrivo.zyking.xyz/api/health/db
```

成功后应返回 200。
