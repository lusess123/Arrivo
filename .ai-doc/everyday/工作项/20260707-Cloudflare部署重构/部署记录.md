# Cloudflare 部署重构记录

## 目标

把 Arrivo 从本地 Nest 服务和本地音频文件缓存，迁到可部署在 Cloudflare 的结构：

- API 使用 Hono Worker。
- 音频文件缓存使用 Cloudflare R2。
- Edge TTS 直接从 Worker 调线上 WebSocket 服务，不再跑 Python。
- 数据库继续使用现有线上 PostgreSQL，不切 D1。
- 用户端和管理端使用 Cloudflare Workers Static Assets 托管。

## 已完成

- 新增 `apps/api` Worker API，支持 `/auth`、`/article`、`/tts`、`/mdd`。
- 新增 `packages/arrivo/contracts`、`packages/arrivo/application`、`packages/arrivo/db`、`packages/arrivo/runtime`，把共享契约、业务用例、数据库适配、Worker 环境边界拆开。
- TTS 音频生成已改成 Worker 直接调用 Edge TTS WebSocket，并缓存到 R2 bucket `arrivo-audio-cache`。
- 管理端 MDD 类型从旧 `arrivo-server` 迁到 `@arrivo/contracts`，避免静态前端继续依赖 Nest 包。
- 前端和管理端都增加生产 API 地址配置，线上默认调用 `https://api-arrivo.zyking.xyz`。
- 新增 Workers Static Assets 配置：
  - `packages/arrivo-fe/wrangler.jsonc` -> `https://app-arrivo.zyking.xyz`
  - `packages/arrivo-manage/wrangler.jsonc` -> `https://manage-arrivo.zyking.xyz`
- Worker API 已部署到 `https://api-arrivo.zyking.xyz`。
- 用户端和管理端静态站已部署到 Cloudflare。

## 历史阻塞和解决

最初数据库网络没有打通。

现有本地 `.env` 的 PostgreSQL 地址是 `111.229.179.154:5433`。验证结果：

- 本机 TCP 连接该地址超时。
- Cloudflare Hyperdrive 创建连接时超时。
- Worker 线上 `/health/db` 约 20 秒后返回 500。
- Worker 日志中的真实错误是 `proxy request failed, cannot connect to the specified address`。
- `whois` 显示 `111.229.179.154` 网段属于 TencentCloud。
- 当前阿里云 CLI RAM 子账号没有 `rds:DescribeDBInstances` 和 `ecs:DescribeInstances` 权限，所以无法直接查询或修改安全组/白名单。
- `111.229.179.154` 只有 SSH 22 端口能从本机连通，`80/443/3000/3025/3026/3027/5432/5433` 都超时。
- `~/.ssh/config` 中有 `111.229.179.154`，但当前 known_hosts 指纹与远端不一致；使用临时 known_hosts 后，现有候选 key 和常见云主机用户都无法登录。
- 本机没有可用腾讯云 CLI 凭证；CloudBase 只有登录态结构，未提供可操作 CVM/RDS 权限。
- Cloudflare Tunnel 私网数据库方案要求 `cloudflared` 运行在能访问数据库的私网中。

后续参考 `edu-tree`，确认可复用同一台阿里云服务器上的 Cloudflare Tunnel，并在该服务器上创建 Arrivo 独立 PostgreSQL 容器，最终改成 `Hyperdrive -> Workers VPC Service -> Tunnel -> 127.0.0.1:5435 -> arrivo-postgres`。

## R2 费用判断

Cloudflare R2 Standard 有免费额度，适合当前音频缓存。已按 Cloudflare 官方 R2 pricing 页确认：<https://developers.cloudflare.com/r2/pricing/>

- 10 GB-month storage / month
- 1 million Class A operations / month
- 10 million Class B operations / month
- 直接从 R2 出公网 egress 免费

注意：免费额度只适用于 Standard storage，不适用于 Infrequent Access。

## 2026-07-07 最终上线结果

按 `edu-tree` 的成功路径改为：

```text
Arrivo Worker
  -> Hyperdrive arrivo-postgres-prod
  -> Workers VPC Service arrivo-postgres-prod
  -> Cloudflare Tunnel edu-tree-demo
  -> 阿里云服务器 127.0.0.1:5435
  -> arrivo-postgres
```

重构修正：

- `apps/api` route 不再 import `@arrivo/db`。
- `@arrivo/db` 不再感知 `HYPERDRIVE` 或 Worker env，只接收连接串。
- 新增 `@arrivo/infra`，承接 Edge TTS client 和 R2 audio cache adapter。
- Worker 入口负责把 Hyperdrive、R2、Edge TTS 注入 application runtime。
- `routes/index.ts` 改成 `routes.register.ts`，文件名体现路由组装职责。
- 新 Cloudflare API 链路不使用 Nest.js；旧 `packages/arrivo-server` 只作为 legacy 历史包留存。
- 根脚本中的旧 Nest/旧 DB 构建入口改名为 `legacy-*`，避免误认为默认服务端路径。
- 根 workspace 不再默认包含旧 `packages/arrivo-server`，Cloudflare 默认链路不再使用 Nest.js。
- 手机验证码支持固定码登录，固定码来自 Worker env `FIXED_SMS_CODE` 或数据库 `Config.FixedCode`，业务用例里不散落硬编码。
- 用户端登录成功后改成浏览器级跳转，避免 SPA 状态时序导致登录成功后又回到 `/login`。
