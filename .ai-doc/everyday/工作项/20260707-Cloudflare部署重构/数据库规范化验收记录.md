# 数据库规范化验收记录

## 背景

本次重构按 `edu-tree` 的数据库规范补齐 Arrivo 的 Cloudflare API 数据层：

- 所有业务表统一固定字段：`id / deletedAt / createdAt / updatedAt / deletedBy / createdBy / updatedBy / tenantId / teamId / env`。
- `id` 改为 PostgreSQL `uuid`，默认值为 `uuidv7()`。
- `tenantId` 默认 `helloworld`，查询、写入、更新、删除都进入租户作用域。
- 默认软删除，业务查询默认过滤 `deletedAt = null`。
- 自然唯一键和常用查询索引按 `tenantId` 作用域建立。

## 代码改动

- 新 `@arrivo/db` 包补齐旧迁移历史，并新增迁移：
  - `packages/arrivo/db/prisma/migrations/20260707160000_normalize_schema/migration.sql`
- `packages/arrivo/db/prisma/schema.prisma` 已改为 uuid(7)、固定字段、租户索引和唯一约束。
- `packages/arrivo/application/src/runtime/data-scope.ts` 集中提供租户、软删除、uuidv7、审计字段生成逻辑。
- Auth、Article、MDD 用例已改为：
  - 查询默认带 `tenantId` 和 `deletedAt = null`
  - 创建显式使用 uuidv7
  - 更新/删除带租户条件
  - MDD 删除改为软删除

## 线上迁移

- 线上数据库：阿里云服务器上的 `arrivo-postgres` 容器，数据库名 `arrivo`。
- 迁移前备份：
  - `/root/arrivo-backups/arrivo-before-db-normalize-20260707160238.sql.gz`
- 迁移方式：
  - 先在真实库事务中执行完整 migration 并 `ROLLBACK`。
  - dry-run 通过后正式执行 `BEGIN -> migration SQL -> 写入 _prisma_migrations -> COMMIT`。
- 新 migration 记录：
  - `20260707160000_normalize_schema`

## 数据库验收

已确认：

- `public.uuidv7()` 存在并可生成 uuidv7。
- 7 张表的 `id` 均为 `uuid`，默认值为 `uuidv7()`。
- `createdAt` / `updatedAt` 均非空。
- `tenantId` 均非空，默认值为 `helloworld`。
- `deletedBy / createdBy / updatedBy / teamId` 均已改为 `uuid`。
- `Config / UserPassword / PhoneCode / EmailCode` 已补齐 `env`。
- `Articles.userId`、`Sentences.articleId`、验证码 `userId` 已改为 `uuid`。
- 迁移后 tenant 回填为 0 缺失。
- `Articles -> User`、`Sentences -> Articles` 的 tenant 一致性检查为 0 mismatch。
- 关键索引和唯一约束已存在：
  - `User_tenantId_phoneNumber_key`
  - `Articles_tenantId_userId_title_key`
  - `Config_tenantId_key_key`
  - `Sentences_tenantId_articleId_createdAt_idx`
  - `PhoneCode_tenantId_toPhoneNumber_createdAt_idx`
