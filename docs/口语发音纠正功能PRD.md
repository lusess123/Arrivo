## 一、产品概述

### 1.1 背景

Arrivo是一款语言学习软件，致力于帮助用户突破语言障碍。当前市场上的口语发音纠正产品存在以下问题：

| 问题 | 现状 |
|------|------|
| 非实时 | 都是"录完再分析"，无法真正对话 |
| 过于严格 | 苛求完美口音，打击用户信心 |
| 反馈单一 | 只有分数，不知道怎么改 |

### 1.2 产品定位

为英语学习者提供**实时对话+精准分析**的口语发音纠正体验。

### 1.3 核心价值

| 差异点 | 说明 |
|--------|------|
| 边聊边纠 | 对话中实时反馈，不是录完再分析 |
| 只纠真错 | 区分"错误"和"口音"，不吹毛求疵 |
| 精准报告 | 对话后提供音素级详细分析 |

## 二、目标用户

### 2.1 用户画像

| 用户群 | 特征 | 核心诉求 |
|--------|------|----------|
| **职场人士** | 25-40岁，需要英语工作沟通 | 能被理解，不追求完美口音 |
| **留学/移民** | 准备雅思/托福，或海外生活 | 应试+日常交流 |
| **学生党** | 大学生/研究生，英语考试需求 | 提分+口语自信 |

### 2.2 优先级

**MVP阶段优先服务：职场人士**

理由：
- 付费意愿强
- 需求明确（能被理解即可）
- 与Arrivo现有用户群匹配

## 四、核心功能

### 4.1 实时口语对话

#### 功能描述

用户与AI进行英语对话，AI在对话中实时纠正严重发音错误。

#### 用户故事

> 作为一个职场英语学习者，我希望能和AI进行自然的英语对话，当我发音有严重问题时得到即时纠正，这样我可以在交流中提升口语。

#### 交互流程

```
1. 用户进入"口语对话"
2. 选择场景（如：工作汇报、日常闲聊、面试模拟）
3. 开始对话
   - 用户说话 → AI实时回应
   - 发现严重错误 → AI温和纠正
   - 继续对话...
4. 用户结束对话
5. 显示"正在分析..."
6. 展示详细发音报告
```

#### 界面原型

```
┌────────────────────────────────────┐
│  工作汇报场景                    ✕  │
├────────────────────────────────────┤
│                                    │
│  AI: How was your project          │
│      progress this week?           │
│                                    │
│  ──────────────────────────────    │
│                                    │
│  You: I sink we made good progress │
│                                    │
│  AI: Great to hear! By the way,    │
│      I noticed "sink" - did you    │
│      mean "think"? Try putting     │
│      your tongue between teeth. 😊 │
│                                    │
│  ──────────────────────────────    │
│                                    │
│         🎤 按住说话                 │
│                                    │
│  [结束对话]                        │
│                                    │
└────────────────────────────────────┘
```

#### 业务规则

| 规则 | 说明 |
|------|------|
| 对话时长 | 单次最长5分钟（成本控制） |
| 纠错策略 | 只纠正影响理解的错误，不纠口音 |
| 纠错频率 | 同一错误最多提醒2次，避免打扰 |
| 纠错方式 | 融入对话，温和提示 |

#### 错误分级

| 级别 | 示例 | 处理方式 |
|------|------|----------|
| **严重** | think→sink, sheet→shit | 立即纠正 |
| **中等** | 重音位置错 | 对话后报告 |
| **轻微** | r不卷舌 | 不扣分 |

### 4.2 发音分析报告

#### 功能描述

对话结束后，生成详细的发音分析报告。

#### 用户故事

> 作为用户，我希望对话结束后能看到详细的发音分析，知道哪些音素有问题、怎么改进，这样我可以针对性练习。

#### 报告内容

```
┌────────────────────────────────────┐
│  发音分析报告                       │
├────────────────────────────────────┤
│                                    │
│  总体评分: 78/100                   │
│  ████████░░                        │
│                                    │
│  ─────────────────────────────     │
│                                    │
│  📊 维度评分                        │
│  · 准确度: 75  ████████░░          │
│  · 流利度: 82  █████████░          │
│  · 语调:   80  ████████░░          │
│                                    │
│  ─────────────────────────────     │
│                                    │
│  ⚠️ 需要改进                        │
│                                    │
│  1. /θ/ 音（th）                   │
│     错误: think → [sink]           │
│     出现: 3次                       │
│     建议: 舌尖放在上下齿之间        │
│     [🔊 听示范] [🎯 去练习]          │
│                                    │
│  2. 词尾辅音                        │
│     错误: walked → [walk]          │
│     出现: 2次                       │
│     建议: 注意发出词尾的 /t/ 音     │
│     [🔊 听示范] [🎯 去练习]          │
│                                    │
│  ─────────────────────────────     │
│                                    │
│  ✅ 做得好                          │
│  · 语速适中                         │
│  · 句子完整                         │
│                                    │
│  [分享报告]  [再练一次]             │
│                                    │
└────────────────────────────────────┘
```

#### 报告维度

| 维度 | 说明 | 来源 |
|------|------|------|
| AccuracyScore | 发音准确度 | Azure |
| FluencyScore | 流利度 | Azure |
| ProsodyScore | 语调/重音 | Azure |
| 问题音素 | 具体哪些音发错 | Azure |
| 改进建议 | 怎么改 | LLM生成 |

### 4.3 跟读练习（辅助功能）

#### 功能描述

针对报告中的问题音素，提供专项跟读练习。

#### 交互流程

```
1. 用户点击报告中"去练习"
2. 进入针对性练习
   - 显示目标单词/句子
   - 播放示范发音
   - 用户跟读
   - 即时评分反馈
3. 重复练习直到达标
```

### 5.2 技术选型

| 组件 | 技术方案 | 理由 |
|------|----------|------|
| 实时对话 | GPT-4o Realtime API | 160ms延迟，真正对话体验 |
| 发音评测 | Azure Pronunciation Assessment | 专业级音素评分，开箱即用 |
| 报告生成 | GPT-4o-mini | 低成本，生成改进建议 |
| 离线备选 | Whisper + 自建模型 | 降级方案 |

### 5.3 数据流

```
对话阶段:
User Audio ──→ GPT-4o Realtime ──→ AI Response
     │
     └──→ Audio Buffer (同步录制)

分析阶段:
Audio Buffer ──→ Azure Assessment ──→ Scores + Errors
                                           │
                                           ▼
                                    GPT-4o-mini ──→ Report
```

| 场景 | 方案 |
|------|------|
| API不可用 | 降级到离线模式（Whisper） |
